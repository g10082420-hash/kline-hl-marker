# K線頭部/底部自動標記網頁版

## 功能

這版同時包含：

1. 方案1：H / L 模式
   - 頭部標 H
   - 底部標 L
   - H 紅色，L 綠色，外框白色圓形

2. 方案2：頭 / 底 模式
   - 頭部標「頭」
   - 底部標「底」

3. 加強功能
   - 圈出被判定的K棒
   - 顯示標記明細
   - 主圖裁切檢查
   - 可手動輸入價格軸與日期範圍，估算標記日期/價格

4. v12 H / L 定義
   - 突破 5MA：昨 close < 5MA 且今 close > 5MA
   - 跌破 5MA：昨 close > 5MA 且今 close < 5MA
   - 假設今日為 T，每輪從 T 往前找 a = 第一個跌破 5MA 之日，b = 第一個突破 5MA 之日
   - 若 a 早於 b，代表跌破到突破，取 a 與 b 之間的最低 low 標 L
   - 若 a 晚於 b，代表突破到跌破，取 a 與 b 之間的最高 high 標 H
   - 完成一段後，把 T 移到該段左側事件，繼續往左遞推，直到前面沒有成對事件
   - 以上區間都含兩端，同高 / 同低取較右邊
   - K 棒偵測會排除同一欄位中與主要 K 棒分離的圖例 / 標籤色塊，避免 H/L 標到非 K 棒物件
   - K 棒偵測會依日 K 固定間距補齊漏偵測的寬色塊與白 / 灰十字 K，讓手動左右修正不容易跳格

5. v6.1 黑底圖修正
   - 偵測 K 棒時會先扣除橘色 5MA 像素
   - 避免 5MA 被誤抓成紅K，導致多根 K 棒被串成一大段後整段漏判

6. 手動標註模式
   - 保留自動標記結果
   - 可在「手動標註」頁籤選擇目前目標：突破點、跌破點、H、L
   - 點擊主圖中的 K 棒後會吸附到最近 K 棒並建立標記
   - 可選擇以自動標記圖或完全原圖作為手動標註底稿
   - 點選後若吸附到隔壁 K 棒，可用「往左一根 / 往右一根」修正最後一筆標記
   - 每一種目標都可以重複標記
   - 可復原上一筆、清空手動標註、下載手動標註圖

## 本機執行

```bash
pip install -r requirements.txt
streamlit run app.py
```

## iPhone 使用方式

本機或雲端部署後，用 iPhone Safari 開啟網址，上傳截圖即可。

## Streamlit Cloud 部署步驟

### 1. 建立 GitHub repo

建立一個新的 GitHub 專案，例如：

```txt
kline-hl-marker
```

把以下兩個檔案放進去：

```txt
app.py
requirements.txt
```

### 2. 登入 Streamlit Cloud

到 Streamlit Cloud 登入，使用 GitHub 帳號授權。

### 3. 新增 App

選：

```txt
New app
```

設定：

```txt
Repository: 你的 kline-hl-marker repo
Branch: main
Main file path: app.py
```

### 4. Deploy

按下 Deploy。

完成後會得到一個網址，例如：

```txt
https://你的app名稱.streamlit.app
```

之後 iPhone 直接用 Safari 開這個網址即可。

## 使用提醒

### 主圖裁切

如果偵測失敗，最常是主圖範圍沒抓好。

請到左側調整：

- 主圖上緣
- 主圖下緣
- 主圖右邊界

橘框應該只框住主K線圖，不要框到成交量、KD、MACD。

### 日期/價格明細

目前沒有做OCR，所以日期與價格是估算。

若要明細顯示日期/價格，請開啟「啟用估算日期/價格」，手動輸入：

- 主圖上緣價格
- 主圖下緣價格
- 左側第一根K棒日期
- 右側最後一根K棒日期

這樣程式會依照K棒位置估算日期與價格。
